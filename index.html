Hi, we're Grapple.

We think everyone should be able to boulder, when it suits them. At Grapple we aim to solve some of the problems currently associated with bouldering to bring it to everyone.

Far Locations

Climbing centres are great, but the large area required means they can't be situated in city centres. The fact that climbing centres can be difficult to reach may be offputting to many climbers, who may be more inclined to climb if centres were easier to access.

Fixed Routes

The routes available to climb at climbing centres are typically set by climbing centre staff, and can remain on a wall for up to a month depending on how frequently the staff update the routes. Once you solve as many routes as you can, climbers need to wait for route updates to try new problems.

Non-Automation

Changing climbing routes requires an experienced staff member to manually remove and replace each hold, iteratively testing the route as they go. This can consume a lot of time and manpower.

Website Creation Tools

This website was created using a HTML/CSS/JavaScript template available to download free from Bootstrap. Bootstrap templates are useful because most of the aesthetic and functional portions of the website have already been set up, so only the content needs to be altered to suit the specific application. The source code content of the website was modified using the Atom source code editor, and code changes were tracked using GitHub Desktop. The combination of Atom and GitHub desktop results in an efficient workflow that can edit and push local changes quicky.

Locking Mechanisms

Climbing holds can be locked into place by fixing or unfixing the rotation of the lead nut, achieved using a servo motor mechanism.

Climbing holds are positined on carriages that can move linearly along linear shafts behind the wall. Up to three holds can be positoned on each set of rails.

What Makes a Good Website?

A good website delivers a clear message in an engaging way. This can be achieved through:

  • Easily comprehendable information - the main content of the website should be relatively easy to understand at first glance, to avoid confusion or miscommunication.

  • A simple navigation system. This allows the user to reach all locations of the website with ease and by using intuitive interactive features.

  • An aesthetically pleasing layout that draws users to the website and keeps them engaged throughout.

  • Relevant and informative graphics that complement the main content without distracting from it.

  • Lack of overcrowding. Allowing the website some 'breathing space' when positioning the elements means that the useful information is easy to identify when the website is looked at as a whole.

Accessibility Considerations

Making a website accessible means that understanding the intended website message is not hindered by the presence of disabilities, such as visual or auditory impairment. It also allows the website to be viewed on a range of devices without hindering the experience.

Methods of achieving website accessibility include:

  • Utilising the 'alt' alternative information tag for images

  • Using a suitable contrast ratio for between text and background colours

  • Minimising the use of unskippable flashing or blinking content

GDPR

The General Data Protection Regulation (GDPR) fully came into force the European Union in May 2018 with the intention of protecting information on the internet. It is a replacement of the Data Protection Act of 1998 (known as the Data Protection Directive outside the UK) in order to account for the rise in internet usage.

Considerations that should be taken for a website to comply with GDPR include:

  • A clear statement of the privacy policy, which assures the user that their data will not sold to third parties without prior consent

  • Minimising the amount of data about the user that is collected through form submissions

  • Gaining consent to use cookies that track user information

  • Using web plugins that comply with GDPR - for example, sending a web page by email should not store the user's email address without prior consent
